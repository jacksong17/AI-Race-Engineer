FIVE TECHNICAL GEMS TO DROP
----------------------------

1. TYPE SAFETY (race_engineer.py:17)
   "TypedDict gives compile-time checking with runtime flexibility.
   Optional[] fields populate as state flows. IDE autocomplete, mypy
   validation, self-documenting contracts."

2. FEATURE SCALING (race_engineer.py:100-103)
   "StandardScaler normalizes to zero mean, unit variance. Without it,
   PSI (25-35) and weight (50-60%) have incomparable coefficients.
   After scaling, -0.08 vs -0.03 means 2.7x more important."

3. CONDITIONAL ROUTING (race_engineer.py:176)
   "First-class error handling at graph level. If telemetry fails,
   route to error node, not crash. In CrewAI, I'd need manual try-catch
   everywhere. LangGraph makes failure modes structural."

4. GRACEFUL DEGRADATION (ibt_parser.py:17)
   "Import-level feature flags. With pyirsdk: parse real 12MB binary files.
   Without: use mock data. One codebase works in prod, CI/CD, and demos.
   Inspired by feature flags but at import level."

5. WHY LANGGRAPH (30 seconds)
   "Evaluated LangGraph, CrewAI, AutoGen. Deciding factor: determinism.
   Same input must give same output for safety. LangGraph's explicit
   state graph guarantees this. CrewAI/AutoGen use LLMs which vary.
   Great for creative tasks, wrong for numerical optimization."

THREE LIMITATIONS TO MENTION
-----------------------------
1. "No confidence intervals - need bootstrap resampling for uncertainty"
2. "Linear only - would add polynomial features for interactions"
3. "Batch processing - real-time would need StreamingAgent"

QUESTIONS & ANSWERS
-------------------

Q: "How would you handle hyperparameter tuning?"
A: "Add HyperparameterAgent before AnalysisAgent. Grid search with
   5-fold CV, cache results. For more params, Bayesian with Optuna."

Q: "What about overfitting with small data?"
A: "20 samples, 8 features = risky. Mitigation: L2 regularization (Ridge),
   feature selection (LASSO), cross-validation (LOOCV), domain constraints."

Q: "How do you version models?"
A: "LangGraph checkpointing + MLflow. Add 'model_version' to TypedDict,
   pickle artifacts with semantic versions, A/B test, keep last N for rollback."

Q: "Why not neural nets?"
A: "Three reasons: Data efficiency (need 100s), interpretability (linear
   coefficients explain why), reliability (deterministic). Would try NN
   with 500+ sessions, but linear is 80% of value."

Q: "Why not just use a notebook?"
A: "Notebooks great for exploration, poor for production. No state mgmt,
   no reusability, can't unit test, no orchestration. Agents give modular,
   testable, CI/CD-friendly components. Explored in notebook, productionized
   as agents."
